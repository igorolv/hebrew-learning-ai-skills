# Пополнение HEBREW_HARRY_POTTER_MASTER_STYLE

## 1. Когда запускается процесс

При обработке каждого нового фрагмента текста (обычно ≈6 страниц). Пополнение MASTER_STYLE выполняется **после** прочтения нового фрагмента и **перед** созданием концептов иллюстраций.

---

## 2. Входные данные

- Новый фрагмент текста (страницы N–M)
- Текущая версия `HEBREW_HARRY_POTTER_MASTER_STYLE.md`

### Образец структуры

В папке `references/` этого skill лежит `HEBREW_HARRY_POTTER_MASTER_STYLE_example.md` — пример заполненного стайлгайда. Прочитай его через `view` перед первым использованием, чтобы понять структуру секций (IV, IX, X, XI).

---

## 3. Алгоритм (5 шагов)

### Шаг 1: Анализ нового фрагмента

Прочитать текст и составить списки:
- **Новые персонажи** — кто появляется впервые?
- **Новые локации** — какие места описаны впервые?
- **Существующие персонажи в новых ситуациях** — кто из уже описанных персонажей появляется, и отличается ли их визуал от эталона?

### Шаг 2: Добавить новых персонажей → секция IV

Для каждого нового персонажа создать запись в **секции IV «Персонажный эталон»**. Формат — таблица с параметрами:

```markdown
### Имя Персонажа

| Параметр | Описание |
|---|---|
| **Фигура/Телосложение** | ... |
| **Лицо** | ... |
| **Волосы** | ... |
| **Одежда (базовая)** | ... |
| **Поза/Пластика** | ... |
| **Настроение** | ... |
```

**Правила:**
- Описывать только визуальные характеристики, релевантные для иллюстрации
- Базовая одежда — та, в которой персонаж появляется чаще всего или впервые
- Если персонаж пока описан мало (мельком упомянут) — можно ограничиться краткой записью со списком, как у Дедалуса Дигла
- Эталон фиксируется **один раз** и не меняется. Все дальнейшие изменения — в секции X

### Шаг 3: Добавить новые локации → секция IX

Для каждой новой локации создать запись в **секции IX «Локации»**. Формат:

```markdown
### Название локации

| Параметр | Описание |
|---|---|
| **Уровень магии** | 1 / 2 / 3 (по шкале из раздела VI) |
| **Палитра** | ... |
| **Освещение** | ... |
| **Ключевые детали** | ... |
| **Настроение** | ... |
```

**Правила:**
- Уровень магии обязательно привязывать к шкале из раздела VI
- Если локация имеет разные состояния (день/ночь, первое/повторное посещение), описывать как отдельные подварианты или указывать диапазон
- Палитра должна оставаться в рамках общего стиля (раздел II) — никаких кислотных цветов, высокого контраста и т.д.

### Шаг 4: Добавить сцены → секция X

Для каждого персонажа, появляющегося в новом фрагменте, добавить запись в **секции X «Сцены и ситуации»** под его именем. Формат:

```markdown
#### Гл. {номер}, стр. {N–M} · Краткое название сцены

| Параметр | Описание |
|---|---|
| **Локация** | Название из секции IX |
| **Уровень магии** | 1 / 2 / 3 |
| **Одежда** | Эталон / описание отличий |
| **Эмоциональный тон** | ... |
| **Отличия от эталона** | Нет / конкретные отличия |
```

**Правила:**
- Сцены располагаются хронологически внутри каждого персонажа
- Если персонаж новый — сначала создать эталон (шаг 2), затем добавить сцену
- Поле «Отличия от эталона» — самое важное. Именно здесь фиксируется то, что генератору иллюстрации нужно знать помимо базового описания
- Если ничего не отличается от эталона — писать «Нет» или «Эталон»

### Шаг 5: Обновить лог → секция XI

Добавить строку в таблицу лога:

```markdown
| {дата} | Глава {N} | {страницы} | Краткое описание: что добавлено |
```

---

## 4. Чеклист перед завершением

- [ ] Все новые персонажи добавлены в секцию IV?
- [ ] Все новые локации добавлены в секцию IX?
- [ ] Для каждого персонажа в новом фрагменте есть запись в секции X?
- [ ] Уровни магии в сценах и локациях соответствуют шкале из раздела VI?
- [ ] Нет конфликтов с разделами I–III (стиль, философия, запреты)?
- [ ] Лог обновлён?

---

## 5. Что НЕ нужно менять

При пополнении **никогда** не редактируются:
- Раздел I (Философия проекта)
- Раздел II (Общий художественный стиль)
- Раздел III (Абсолютные запреты)
- Раздел V (Правила магии)
- Раздел VI (Уровни проявления магии)
- Раздел VII (Шаблон промпта)
- Раздел VIII (Диагностика ошибок)
- Уже зафиксированные эталоны в разделе IV (только дополнение новых персонажей)

## 6. Пример: обработка нового фрагмента

**Задание:** обработать главу 2, стр. 36–41.

**Шаг 1 — анализ:**
- Новые персонажи: Гарри Поттер (10 лет), Дадли Дурсль
- Новые локации: чулан под лестницей
- Существующие персонажи: Вёрнон (утро, раздражён), Петуния (кухня, суетится вокруг Дадли)

**Шаг 2 — новые персонажи в IV:**
Добавить эталоны Гарри и Дадли.

**Шаг 3 — новые локации в IX:**
Добавить «Чулан под лестницей» с палитрой, освещением, уровнем магии.

**Шаг 4 — сцены в X:**
- Под «Гарри Поттер» — первая сцена: пробуждение в чулане
- Под «Дадли Дурсль» — первая сцена: утро дня рождения
- Под «Вёрнон Дурсль» — новая сцена: утро за завтраком (отличия от эталона: домашняя одежда вместо костюма)
- Под «Петуния Дурсль» — новая сцена: суетится вокруг Дадли

**Шаг 5 — лог:**
```
| 2025-XX-XX | Глава 2 | 36–41 | Персонажи: Гарри, Дадли. Локация: чулан. Сцены: утро Дадли, пробуждение Гарри |
```
