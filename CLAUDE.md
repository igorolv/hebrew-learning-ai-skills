# CLAUDE.md

Инструкции, специфичные для Claude. Общие инструкции — в [AGENTS.md](AGENTS.md).

## Как работать со skill'ами

Каждый skill описан в `SKILL.md` внутри своей папки. Перед выполнением:

1. Прочитай `SKILL.md` целиком
2. Прочитай файлы из `references/`, если skill на них ссылается
3. Следуй инструкции пошагово

## Контекст использования

Этот репозиторий используется двумя способами:

- **Claude.ai (Projects)** — файлы `SKILL.md` и `references/` добавляются в Project Knowledge вместе с входными материалами
- **Claude Code** — skill'ы подключаются как локальные папки; можно запускать pipeline'ы последовательно

## Выполнение pipeline'ов

При запуске pipeline'а (нескольких skill'ов подряд):

- Выход одного skill'а — вход следующего. Проверяй, что формат и имя файла соответствуют ожиданиям следующего шага.
- Между шагами показывай результат пользователю и жди подтверждения, если в SKILL.md не указано иное.

## Работа с ивритом

- При визуальном чтении изображений (hp-extraction, hebrew-lesson-extract) — копировать текст максимально точно, включая все видимые огласовки.
- Не «исправлять» и не «нормализовать» огласовки, если skill явно не требует этого (hp-translate добавляет огласовки — это единственное исключение).
- Кавычки в ивритском тексте: ״...״ (гершаим), не "...".

## Генерация DOCX

Skill'ы `*-docx` генерируют Word-документы через Python-скрипты. При выполнении:

- Устанавливай зависимости (`python-docx` и др.) через `pip install --break-system-packages -q`
- Проверяй, что шрифты для иврита (David) доступны в системе
